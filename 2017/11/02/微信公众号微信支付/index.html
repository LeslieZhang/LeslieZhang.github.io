微信公众号内微信支付

发表于 2017-11-02 | 分类于 微信网页开发
微信支付配置信息，JSAPI接口，H5调用微信JS-SDK接口支付，微信公众号支付...

刚做了个微信公众开发的项目，里面有到微信授权登录和微信支付，现在总结一下哈。
微信支付这块微信官方给的文档真是给的十分操蛋，很多都不详细我接手的时候，微信的公众号什么的，都已经申请下来了，微信支付的服务也已经申请下来了。(微信支付功能需要去微信支付，商户平台申请)
参考文档：微信公众号平台-微信JS-SDK说明文档

1、配置一些东西，当然这都是一些最基本的东西

(1)、公众号信息，AppID、AppSecret
(坑点：这个ip白名单，在开发阶段需要填写服务器的ip地址的)

(2)、进入“公众号设置”→“功能设置”→填写“JS接口安全域名”

(3)、上面说了需要去微信支付里的商户平台申请，申请之后也要去拿信息
(这里的授权目录，一定要在你支付页面上，不然支付的页面没有权限去支付了，注意：大小写千万不能错哦)

(4)、微信支付授权目录填写规则
(非常坑的就是这个，因为那个填写的地方没说清楚。)
注意到第三点：
3、头部要包含http或https，须细化到二级或三级目录，以左斜杠“/”结尾

这里非常容易误解。很有可能是这是错误的文案，微信没有及时更新说明文案。
经过实际测试后发现，第三条的意思是支付授权目录应该为当前页面链接的上一级目录

例如：我们的微信支付页面：http ://www.zhangfu.online/order/pay.html，
那么我们的支付授权目录应该配置为http ://www.zhangfu.online/order/
带参数的url忽略参数：
微信支付页面：http ://www.zhangfu.online/order/pay.html?package=1
授权目录应为：http ://www.zhangfu.online/

2、JSSDK使用步骤

(1)、绑定域名
先登录微信公众平台进入“公众号设置”的“功能设置”里填写“JS接口安全域名”。
备注：登录后可在“开发者中心”查看对应的接口权限。
(2)、引入JS文件”
在需要调用JS接口的页面引入如下JS文件，（支持https）：http://res.wx.qq.com/open/js/jweixin-1.2.0.js

备注：支持使用 AMD/CMD 标准模块加载方法加载
(3)、通过config接口注入权限验证配置

(4)、通过ready接口处理成功验证，使用微信支付接口

(5)、通过error接口处理失败验证

备注：prepay_id 通过微信支付统一下单接口拿到，paySign 采用统一的微信支付 Sign 签名生成方法，注意这里 appId 也要参与签名，appId 与 config 中传入的 appId 一致，即最后参与签名的参数有appId, timeStamp, nonceStr, package, signType。
